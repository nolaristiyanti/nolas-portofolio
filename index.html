Tes ADHD

AI-Assisted Early Screening Tool for Children

✕

Summary

A web-based behavioral screening application designed to help parents identify early signs of ADHD in children aged 4–12. The system implements a fuzzy logic algorithm to analyze symptom patterns based on DSM-5 criteria and generate decision-support results. This tool is intended for early screening — not clinical diagnosis.

Problem

  • Early ADHD symptoms are often mistaken as normal childhood behavior.
  • Delayed awareness leads to late intervention and support.
  • There is a lack of accessible digital screening tools for parents.

Goals

  • Develop a web-based ADHD screening system.
  • Translate DSM-5 behavioral criteria into measurable inputs.
  • Implement fuzzy logic for symptom-based inference.
  • Provide clear, understandable screening results.

Scope

  • Screening tool only (not a diagnostic system).
  • Target users: parents/guardians of children aged 4–12.
  • Assessment divided into Activity and Attention sections.
  • Result classification based on fuzzy inference model.

Roles & Tools

  • System analysis & requirement definition
  • Fuzzy algorithm design & implementation
  • Web app development : PHP, HTML, CSS (Bootstrap), MySQL
  • System testing & validation with pediatric expert

Process

  • Researched DSM-5 ADHD criteria and behavioral indicators.
  • Designed fuzzy membership functions and rule base.
  • Implemented web-based questionnaire and scoring logic.
  • Conducted alpha testing, usability testing, and expert validation.

LMS Data Architecture Project

Database Design & SQL Implementation for LMS

✕

Description

A database-focused mini project designed to demonstrate understanding of relational database design and SQL fundamentals using a Learning Management System (LMS) case study. The project covers ERD modeling, MySQL schema design, data manipulation, and complex query operations including joins and aggregation.

What I Did

  • Designed an Entity Relationship Diagram (ERD) for a simple LMS system.
  • Defined primary keys, relationships, and cardinality between tables.
  • Converted the ERD into a MySQL database schema using DDL (CREATE DATABASE, CREATE TABLE).
  • Implemented proper constraints including PK, FK, NOT NULL, and data types.
  • Populated database with sample data using DML (INSERT INTO with 10–15 records per table).
  • Executed SQL queries ranging from basic SELECT to advanced JOIN and aggregation queries.

Key Focus

  • Database design and normalization (ERD modeling).
  • MySQL schema creation using DDL.
  • Data manipulation using DML.
  • SQL fundamentals (SELECT, WHERE, ORDER BY, LIMIT).
  • Join operations (INNER JOIN, LEFT JOIN)
  • Aggregate functions (COUNT, AVG, GROUP BY, HAVING)
  • Conditional logic in SQL queries

SQL Topics Covered

  • Basic Queries.
  • Aggregate Queries.
  • Join Operations.

Tech Stack

  • MySQL.
  • SQL (DDL, DML, DQL).
  • Database Design (ERD Modeling).

Result

Successfully designed and implemented a relational database for an LMS case study, demonstrating strong understanding of SQL fundamentals, relational modeling, and query optimization techniques from basic operations to advanced joins and aggregation.
